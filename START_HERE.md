# START HERE: SWI Reviewer Guide

## One-Paragraph Explanation
The Secure Witness Interface (SWI) is a security-first runtime for AI agents. It acts as a "cryptographic gatekeeper" that intercepts every action requested by an AI, validates it against a signed policy manifest, and only permits execution if the action is within admissible bounds. Every decision is cryptographically signed and recorded in an immutable audit chain, providing evidence-grade accountability for autonomous systems.

## Component List
- **`app/`**: The frontend monitoring console.
- **`core/kernel/`**: The authoritative decision logic (`SWIKernel`).
- **`core/manifest/`**: The signed policy definition (`manifest.json`).
- **`core/keys/`**: Public key for signature verification. (Private key is generated on-the-fly for local dev but excluded from release).
- **`certification/`**: S9 Proof artifacts and regulatory binding docs.

## How to Run
1. **Install**: `npm install`
2. **Start**: `npm run dev`
3. **Access**: Open `http://localhost:3000`
4. **Test**: Navigate to the "Red-Team" tab to run adversarial simulations.

## Key Logic Locations
- **Manifest**: `core/manifest/manifest.json`
- **Audit/Replay Logic**: `core/kernel/hashChainKernel.ts`
- **S9 Proof Logic**: `core/kernel/s9_proof.ts`
- **Consensus Engine**: `core/kernel/consensus.ts`

## Certification Docs
The `certification/` directory contains S9 Proofs. These are generated by the system to provide continuous evidence that the runtime is operating within the bounds of the Law Binding for Gaborone-based AI Architecture.

---
*Reviewer Note: The system is designed to HALT if any tampering is detected in the manifest or the audit chain.*
