# Contributing to SWI

This repository is dedicated to exploring the concept of **Structured Workflow Intelligence**, focusing on governance-aware AI execution, determinism, and proof-of-work systems.

## The Scope of This Prototype

We welcome contributions specifically toward demonstrating the educational and research goals of the system. This includes, but is not limited to:
- Enhancing the **Formal Verification** and TLA+ specifications.
- Deepening **ZK Proof** examples or integration mockups.
- Improving **Drift Detection mechanisms** and policy execution simulations.

### What is *NOT* inside the Scope

Please **do not** submit pull requests aiming to:
- "Productionize" the code or prepare it for a secure cloud deployment. (It is intrinsically a prototype).
- Remove security constraints, logs, or traces in order to speed up the application. (The traces are the point of the project).
- Integrate direct connections to real financial, high-stakes, or operational endpoints. (Keep it in sandbox mode).
- Propose changes claiming cryptographic readiness or full trustless status.

## Steps to Contribute

1. **Fork the Repository**.
2. **Review `GOVERNANCE_RULES.md` and `README.md`**. Ensure your changes align with the prototype and educational nature of this repository.
3. **Write Clear Evidence-Based Commits**. Follow the system's focus on evidence. Describe not just what you changed, but the rationale.
4. **Submit a Pull Request**. Outline the specific components you modified and mention if you introduced any new simulated crypto/ZK approaches.

## Code of Conduct
We adhere to standard open-source conventions. Be respectful, be curious, and focus on the research.
